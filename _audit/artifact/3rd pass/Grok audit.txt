## PASS 1 — CRITICAL BLOCKERS

### BUG-001 [BLOCKER] — `_parse_robocopy_bytes` not reachable from pre-flight thread
**File:** `ghost_harvest/app.py`  
**What's wrong:** `_thread_preflight` calls `self._parse_robocopy_bytes` but the method is defined as `@staticmethod` *after* the call site in the class body (and the thread does not have access to the bound method anyway). This raises `AttributeError` on pre-flight.  
**Fix:** Move the staticmethod to the top of the `GhostHarvest` class (right after `_update_space`) and ensure it's called as `GhostHarvest._parse_robocopy_bytes(...)`.

```python
# Insert near top of GhostHarvest class (after _update_space)
    @staticmethod
    def _parse_robocopy_bytes(line: str) -> int:
        """
        Parse a robocopy summary 'Bytes' line...
        """
        # (keep the entire existing implementation unchanged)
```

### BUG-003 [BLOCKER] — `scan_plain` not passed to scanner in pipeline
**File:** `ghost_harvest/app.py`  
**What's wrong:** In `_pipeline`, the scanner is instantiated with `scan_plain=settings["scan_plain"]` — but the `PostCopyScanner.__init__` signature in the provided snapshot does **not** accept `scan_plain` (it is a recent addition that was not propagated to the class).  
**Fix:** Update the scanner init in `_pipeline`:

```python
scanner = PostCopyScanner(

```

And update `scanner.py` `__init__` to accept and store it (see PASS 2).

## PASS 3 — MEDIUM SEVERITY (Silent Failures / Latent Issues)

### BUG-007 [MEDIUM] — Hardcoded Windows paths + no cross-platform guard in some places
**File:** `ghost_harvest/app.py`, `main.py`  
**What's wrong:** Several `\\` assumptions and `sys.platform` checks exist, but `_browse_dest` and queue handling do not normalize consistently. On non-Windows it will still attempt to run.  
**Fix:** In `_browse_dest` and `_q_add`:

```python
p = p.replace("/", "\\") if sys.platform == "win32" else p
```

Reinforce top-level guard in `main.py`.

### BUG-008 [MEDIUM] — `build_display_cmd` double-escapes quotes incorrectly
**File:** `ghost_harvest/command.py`  
**What's wrong:** `a = a.replace('"', r'"')` does nothing useful (raw string literal). Should escape for display.  
**Fix:**

```python
if '"' in a:
    a = a.replace('"', '\\"')
```

### BUG-009 [MEDIUM] — No handling for abort during SHA-256 verification
**File:** `ghost_harvest/app.py` + `hasher.py`  
**What's wrong:** `_pipeline` checks abort before scan/hash but `ParallelHashVerifier.verify` has no early-exit path. Long-running verify can continue after stop.  
**Fix:** Pass `abort_event` to verifier and check inside the worker loop (simple `if self.abort_event.is_set(): return`).

### BUG-010 [MEDIUM] — `strip_ansi` called on every line but not on pre-flight summary
**File:** `ghost_harvest/app.py`  
**What's wrong:** Minor log pollution.  
**Fix:** Wrap summary_text with `strip_ansi` before logging.
